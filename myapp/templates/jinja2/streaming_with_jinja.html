4) Solution: combine StreamingHttpresponse() with jinja.generate()


Thanks to jinja2.generate(), we have the best of both worlds : the page uses a template, and the content is streamed.

The context dictionary contains a string : {{mystring}}

And also a short list, that will be rendered very quickly by StreamingHttpResponse() :

    {% for n in mylist %}
  • {{n}}
    • {% endfor %}

The next element takes about 5 seconds to render, but the page displays every element as it is processed.

    {% for element in mylongresponse %}
  • {{element}}
    • {% endfor %}
